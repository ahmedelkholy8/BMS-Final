CONFIRMED BUGS AND MISSING CONNECTIONS

**Bug** Panel.Simulation is always visible — blocks the 3D scene
Panel.Simulation is active on startup and covers the entire screen (anchors 0.0 → 1.1). NavBtn.Simulation link to NULL — clicking it does nothing. The new window never hides/show; this panel is the 3D scene is invisible in on launch.

**Bug** SimulationController lives on its own orphan GameObject
It’s on a GO called “SimulationController” floating in the scene root, not on BatteryPanel. SimulationUI is also on a floating GO. Neither is part of Panel.Simulation’s hierarchy, so no nav page show/hide won’t affect them correctly. SimulationController.SetSimPageOpen() is never called.

**Bug** NavController panels array is mostly NULL
Only NavBms, Overview — Panel, Right and NavBtn_History — Panel, History are wired. NavBtn_Simulation — NULL meaning clicking Simulation does nothing. All other nav buttons (Cells, Hardware, Telemetry, Analytics, Alerts, Settings) also have NULL panels.

**Issue** SimulationUI slider refs point to wrong components
The left feeds slider, sliderAmbient etc. in SimulationUI reference the Slider child components inside each row — need to verify all 3 sliders and both dropdowns are correctly wired, and that the severity `slider` maps correctly (the UI built it as a 3rd step, but SimSeverity is a Low/Med/High enum).

**Issue** Visualization panel RenderView shows nothing
The center “Battery Panel” Visualization Image has size 0xD and is not connected to the main camera or a RenderTexture. The 3D scene view isn’t feeding into it. This is the key visual missing from the Simulation panel — the 3D cell view inside the panel.

**Issue** ThermalThumbnail cell_GroupTransform points to BatteryPack, not CellGroup
The secondary thermal camera will orbit around the wrong center. CellGroup is the correct target (the actual cell meshes), not the BatteryPack parent (board load block etc).

**Issue** WorldSpaceLabels: ESP32 label has NULL anchor
The Billboard label “ESP32” has no target and doesn’t exist by that name? The ESP32 label is silently skipped. Minor but should be fixed or the slot removed.

**Visual** Graph LineRenderers at size 0xD — won’t draw
Each Graph cell RectTransform inside GraphPanel has size like “10x154” but the LineRenderer positions are calculated from RectTransform.rect.size — which is only valid after layout, since the rect is zero before layout. The initial rect may be zero. Need to call Rebuild() to a console after layout, or use CanvasForceUpdateCanvas().

**Issue** SceneLightPulse BMS light color is RGB(1,0.405,0) but orange — should shift cooler when temp is high
Mixer tuning at 22°C: The BMS light should be a warmer amber, not orange-red. The current tempFactor calculation has the right structure but the base color is too aggressive at rest.
